## I. INFRASTRUKTUR TEKNOLOGI

### 1. Cloud Computing

#### A. Inti Konsep

Cloud = penggunaan komputer orang lain via internet.
Tujuan: menghilangkan kebutuhan hardware sendiri.

#### B. Mekanisme

User → Internet → Server (data center) → Database → Response

#### C. Komponen

* Server → pemrosesan
* Database → penyimpanan
* Network → koneksi

#### D. Use Case

* Google Drive → storage
* Netflix → streaming
* Canva → rendering design

#### E. Tools

* AWS, GCP, Azure

#### F. Evaluasi

* Skalabel
* Tidak perlu hardware
  – Bergantung internet
  – Biaya jangka panjang

#### H. Perbandingan

Cloud vs Local:

* Cloud: fleksibel, scalable
* Local: kontrol penuh, terbatas

---

### 2. Server, Database, Network

#### A. Inti Konsep

* Server = komputer yang melayani
* Database = penyimpan data
* Network = jalur komunikasi

#### B. Mekanisme

Client → request → server → database → response

#### C. Komponen

* Router → pengarah trafik
* WiFi → media komunikasi
* LAN → jaringan lokal

#### D. Use Case

* WhatsApp messaging system
* Google Docs sync

#### F. Evaluasi

* Sistem scalable
  – Kompleks untuk dipahami awal

---

### 3. Docker & Self-Hosting

#### A. Inti Konsep

Docker = menjalankan aplikasi dalam container terisolasi

#### B. Mekanisme

Host OS → Docker Engine → Container → App

#### C. Komponen

* Image → blueprint
* Container → instance
* Volume → storage

#### D. Use Case

* Nextcloud → personal cloud
* Database server

#### E. Tools

* Docker
* n8n
* Ollama

#### F. Evaluasi

* Portabel
* Konsisten
  – Konsumsi resource
  – Tidak menambah storage

# 3. Sintesis Pengetahuan

## Core Principles

1. **Control vs Passive**

   * Investasi = pasif
   * Bisnis = kontrol

2. **Compounding is Non-Intuitive**

   * Hal kecil → dampak besar

3. **Diversification = Survival**

   * Jangan all-in

4. **Infrastructure = Power**

   * Yang mengontrol sistem → menang
